x86decomp hypothesis create

Creates a new hypotheses record.

Usage

x86decomp hypothesis create [-h] --origin ORIGIN statement scope_kind scope_id

Example

x86decomp hypothesis create recovered-parser-loop function pe-rva:00001000 --origin analyst

Arguments

ArgumentWhat it controls
statement
required		Controls statement for this command.
scope_kind
required		Controls scope kind for this command.
scope_id
required		Identifier of the scope used by the command.
--origin
required		Controls origin for this command.

x86decomp hypothesis dependency

Records a dependency between hypotheses.

Usage

x86decomp hypothesis dependency [-h] hypothesis_id depends_on_id

Example

x86decomp hypothesis dependency hypothesis-001 ./target.exe

Arguments

ArgumentWhat it controls
hypothesis_id
required		Identifier of the hypothesis being operated on.
depends_on_id
required		Identifier of the depends on used by the command.

x86decomp hypothesis evidence

Attaches evidence to the selected hypotheses.

Usage

x86decomp hypothesis evidence [-h] --stance STANCE --weight WEIGHT --kind KIND --group GROUP [--artifact-sha256 ARTIFACT_SHA256] [--details-json DETAILS_JSON] hypothesis_id evidence_id

Example

x86decomp hypothesis evidence hypothesis-001 example-001 --stance support --weight 1.0 --kind analysis --group independent-source

Arguments

ArgumentWhat it controls
hypothesis_id
required		Identifier of the hypothesis being operated on.
evidence_id
required		Identifier of the evidence record.
--stance
required		Controls stance for this command.
--weight
required		Relative evidence weight used by the decision rule.
--kind
required		Category assigned to the record.
--group
required · default: hypothesis		Controls group for this command.
--artifact-sha256Controls artifact sha-256 for this command.
--details-jsonPath to the JSON file containing details data.

x86decomp hypothesis transition

Moves the selected hypotheses through an approved state transition.

Usage

x86decomp hypothesis transition [-h] --reason REASON [--lock] hypothesis_id state

Example

x86decomp hypothesis transition hypothesis-001 active --reason reviewed-change

Arguments

ArgumentWhat it controls
hypothesis_id
required		Identifier of the hypothesis being operated on.
state
required		Requested workflow state.
--reason
required		Controls reason for this command.
--lockControls lock for this command.
