Verification command reference

x86decomp-test

The independent verification harness exposes five commands for configuration, adapter detection, inventory, full runs, and catalog inspection.

current5 commands
verification harness

x86decomp-test adapters

detect adapters and optionally resolve missing ones

Usage

x86decomp-test adapters [-h] [--config CONFIG] [--resolve] [--verbose]

Example

x86decomp-test adapters --config ./x86decomp-test.json

Arguments

ArgumentWhat it controls
--config
default: x86decomp-test.json		Path to the configuration file.
--resolveTry to locate or configure integrations that are currently unavailable.
--verboseShow detailed progress and diagnostic output.

verification harness

x86decomp-test init-config

create a test configuration

Usage

x86decomp-test init-config [-h] --toolkit-root TOOLKIT_ROOT [--output-root OUTPUT_ROOT] [--config CONFIG] [--install-root INSTALL_ROOT] [--non-interactive] [--allow-network] [--allow-install] [--non-strict]

Example

x86decomp-test init-config --toolkit-root . --output-root ./test-results --install-root ./.x86decomp-test-tools --config ./x86decomp-test.json

Arguments

ArgumentWhat it controls
--toolkit-root
required		Root directory of the x86decomp source tree to verify.
--output-root
default: test-results		Directory where run folders, logs, and reports are created.
--config
default: x86decomp-test.json		Path to the configuration file.
--install-rootDirectory reserved for tools managed by the verification harness.
--non-interactiveRun without asking interactive questions.
--allow-networkAllow network access while resolving integrations.
--allow-installAllow installation of missing integrations.
--non-strictRecord unavailable integrations as blocked instead of treating them as failures.
