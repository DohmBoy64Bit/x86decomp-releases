_preexec

Return the legacy resource-limit callback for API compatibility. The production launcher no longer passes this callback to ``Popen`` because Python code executed between fork and exec can deadlock in a process that has already initialized threads. the exec-based resource wrapper applies the same limits in a fresh Python interpreter immediately before ``exec``.
