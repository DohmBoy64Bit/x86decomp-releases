Getting started

From installation to a verified project

Use these steps for a local, authorized analysis workspace. Commands use sample paths that you can replace with your own files.

1. Install the toolkit

python -m pip install x86decomp_toolkit-0.7.4-py3-none-any.whl

For the complete optional analysis and development stack:

python -m pip install 'x86decomp-toolkit[full,pe,dev]'

2. Inspect the interface

x86decomp --help
x86decomp commands
x86decomp commands --owner core

The command catalog can be filtered by capability owner. Every command in the catalog is documented in the command reference.

3. Initialize the unified project data plane

x86decomp project --project ./work init
x86decomp project --project ./work check

4. Initialize a binary-analysis project

x86decomp init ./target.exe ./analysis-project
x86decomp verify-project ./analysis-project

The project keeps the original binary, normalized metadata, function records, evidence, and workflow state in a reproducible directory structure.

5. Inspect the binary

x86decomp inspect-pe ./target.exe
x86decomp disassemble ./target.exe --rva 0x1000 --size 64

6. Configure the independent test suite

x86decomp-test init-config \
  --toolkit-root . \
  --output-root ./test-results \
  --install-root ./.x86decomp-test-tools \
  --config ./x86decomp-test.json

x86decomp-test run --config ./x86decomp-test.json --verbose

The run command creates the configured output directory and a unique run directory containing logs and reports.

7. Verify an extracted source tree

make verify

This compiles the source, validates contracts and source hashes, and executes the exact current test inventory with zero skips.

Safety boundary: Analyze only binaries and systems you are authorized to inspect. Native execution should be isolated and explicitly approved.
