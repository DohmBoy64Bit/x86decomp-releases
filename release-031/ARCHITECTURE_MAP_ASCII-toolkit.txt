This file is a plain-text projection of the implemented architecture. It must be
updated in the same change as the Mermaid map whenever subsystem boundaries,
inputs, trust boundaries, workflow states, adapters, commands, or ownership
rules change.

MATCHING MODE STATE PATH
========================

 [start]
    │
    ▼
 not_started
    │ source candidate recorded
    ▼
 decompiled
    │ selected compiler profile succeeds
    ▼
 compiles
    │ ABI observations accepted
    ▼
 abi_compatible
    │ normalized instruction / CFG threshold met
    ▼
 instruction_similar
    │ exact modeled function bytes match
    ▼
 byte_matched
    │ verified patch or image integration succeeds
    ▼
 image_integrated
    │ relink and target-specific image checks pass
    ▼
 full_relink_validated
    │
    ▼
  [end]

Any active matching state may move to `blocked` with an explicit blocker. A
blocked state may return only through an audited reset or blocker-resolution
transition recorded in project memory.

FUNCTIONAL MODE STATE PATH
==========================

 [start]
    │
    ▼
 not_started
    │ source candidate recorded
    ▼
 decompiled
    │ selected compiler profile succeeds
    ▼
 compiles
    │ ABI observations accepted
    ▼
 abi_compatible
    │ declared concrete tests agree
    ▼
 differentially_validated
    │ bounded symbolic obligations pass
    ▼
 symbolically_bounded
    │ declared integration scenarios pass
    ▼
 integration_validated
    │
    ▼
  [end]

Any active functional state may move to `blocked` with an explicit blocker. A
blocked state may return only through an audited reset or blocker-resolution
transition recorded in project memory.
