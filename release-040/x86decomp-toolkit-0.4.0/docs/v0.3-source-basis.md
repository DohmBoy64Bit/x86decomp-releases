# Version 0.3 source basis and scope

The v0.3 COFF/PE, COMDAT, relocation, TLS, load-configuration, and exception/unwind
implementations are based on the Microsoft PE/COFF specification and Microsoft x64
exception-handling documentation. Microsoft C++ RTTI layout decisions are cross-checked
against LLVM's Microsoft C++ ABI implementation.

Primary references:

- Microsoft PE format specification:
  https://learn.microsoft.com/en-us/windows/win32/debug/pe-format
- Microsoft x64 exception handling:
  https://learn.microsoft.com/en-us/cpp/build/exception-handling-x64
- LLVM Microsoft C++ ABI implementation:
  https://github.com/llvm/llvm-project/blob/main/clang/lib/CodeGen/MicrosoftCXXABI.cpp

The toolkit uses conservative structural checks and explicit bounds. It does not claim
that every compiler version emits identical private metadata layouts, that a heuristic
record is source-level truth, or that a finite validator proves arbitrary-program
equivalence.

## PDB/MSF implementation basis

- LLVM MSF file format: https://llvm.org/docs/PDB/MsfFile.html
- LLVM PDB stream overview: https://llvm.org/docs/PDB/index.html
- LLVM PDB identity stream: https://llvm.org/docs/PDB/PdbStream.html
- LLVM TPI/IPI stream header: https://llvm.org/docs/PDB/TpiStream.html
- LLVM DBI stream and module/source layouts: https://llvm.org/docs/PDB/DbiStream.html

The implementation parses only the bounded fields represented in its report and marks
full CodeView type/symbol record recovery as false.
