ACTUAL MATCHING-STATE PATH

not_started
    │
    ▼
decompiled
    │ function artifact and candidate representation exist
    ▼
compiles
    │ declared compiler emitted the expected artifact
    ▼
abi_compatible
    │ declared ABI checks passed
    ▼
instruction_similar
    │ versioned normalized instruction/CFG comparison passed
    ▼
byte_matched
    │ exact scoped original/candidate bytes
    ▼
image_integrated
    │ hash-gated patch or relink integration passed
    ▼
full_relink_validated
    │ target-specific relink and image checks pass
    ▼
accepted matching evidence

Any active state may enter blocked. Leaving blocked requires a resolved blocker and an audit event.

ACTUAL FUNCTIONAL-STATE PATH

not_started
    │
    ▼
decompiled
    │ function artifact and candidate representation exist
    ▼
compiles
    │ declared compiler emitted the expected artifact
    ▼
abi_compatible
    │ declared ABI checks passed
    ▼
differentially_validated
    │ all declared concrete harnesses pass
    ▼
symbolically_bounded
    │ bounded symbolic obligations pass with model and truncation recorded
    ▼
integration_validated
    │ declared integration scenarios pass in a recorded environment
    ▼
accepted functional evidence

Any active state may enter blocked. Leaving blocked requires a resolved blocker and an audit event.

INDEPENDENCE RULE

A function may be instruction_similar in matching mode and integration_validated in
functional mode. Neither lane automatically promotes the other.

MAINTENANCE CONTRACT

Keep this file synchronized with docs/ARCHITECTURE_MAP.md. Update both whenever
project schemas, job states, target inputs, workflow states, workers, adapters,
templates, validation lanes, release gates, service ownership, or trust boundaries
change. The integrated test suite enforces version, required-plane, state and link
contracts for all toolkit and test-suite architecture maps.
