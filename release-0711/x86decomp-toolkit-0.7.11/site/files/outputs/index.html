    Toolkit outputs¶

    This page catalogs the files generated by the x86decomp toolkit during project initialization, analysis, compilation, relinking, evidence management, and reporting.

    Project initialization¶

    Files created by x86decomp project init:

    File Format Description
    project.json JSON Project configuration (schema v1–3)
    analysis/program.json JSON PE program manifest — parsed binary metadata
    analysis/host.json JSON Host environment snapshot
    analysis/database/analysis.sqlite3 SQLite Analysis database
    work/tasks.sqlite3 SQLite Work queue database
    state/project-state.sqlite3 SQLite Project state database (v3)
    memory/events.jsonl JSONL Memory event ledger (initialized with one event)
    target-pack/target.toml TOML Target pack identity (v3)

    Analysis outputs¶

    Generated during static analysis and PE inspection:

    File Format Description
    analysis/exports/*.json JSON Exported analysis data
    analysis/ghidra/* Various Ghidra project files and data exports
    analysis/mcp-transactions/* JSON Recorded MCP requests/responses
    analysis/recon/* JSON Reconstruction analysis artifacts

    Compilation outputs¶

    Generated by compiler experiments and matched builds:

    File Format Description
    build/compiler/*.obj COFF Compiled object files
    build/compiler/*.asm Text Assembly listings from compilation
    build/compiler/*.json JSON Compilation reports (timing, diagnostics)
    build/cache/* Various Build cache artifacts

    Relink outputs¶

    Generated by linker reconstruction and relinking:

    File Format Description
    build/relink/manifest.json JSON Relink manifest
    build/relink/*.exe PE Relinked executable
    build/relink/*.dll PE Relinked library
    build/relink/*.json JSON Section layout, COMDAT resolution records
    build/relink/*-report.json JSON Relink execution report
    build/patches/* Binary Binary patch files

    Function outputs¶

    Each function gets a subdirectory under functions/ named pe-rva_XXXXXXXX:

    File Format Description
    pe-rva_XXXXXXXX/function.json JSON Function packet manifest
    pe-rva_XXXXXXXX/*.c C Decompiled C source
    pe-rva_XXXXXXXX/*.cpp C++ Decompiled C++ source
    pe-rva_XXXXXXXX/*.asm Assembly Byte-form assembly (default) or annotated
    pe-rva_XXXXXXXX/references.json JSON or JSONL Cross-reference evidence
    pe-rva_XXXXXXXX/compile.json JSON Compilation attempt report
    pe-rva_XXXXXXXX/byte-comparison.json JSON Byte comparison results
    pe-rva_XXXXXXXX/relocation.json JSON Relocation resolution report
    pe-rva_XXXXXXXX/attempt.json JSON Full attempt record including prompt, response, and validation

    Evidence outputs¶

    File Format Description
    evidence/items/*.json JSON Evidence records
    evidence/claims/*.json JSON Claim records
    evidence/files/* Various File-backed evidence artifacts

    Memory event ledger¶

    File Format Description
    memory/events.jsonl JSONL Append-only audit log of project operations

    Each event is a JSON object on its own line:

    {"actor":"x86decomp","category":"project","summary":"Initialized a native x86 PE decompilation project...","details":{...},"recorded_at":"2026-07-01T12:00:00Z"}

    Pipeline outputs¶

    File Format Description
    orchestration/pipelines/*.json JSON Pipeline manifests
    orchestration/logs/* Text Pipeline stage execution logs
    orchestration/work/* Various Pipeline work artifacts

    LLM outputs¶

    Generated by local LLM matching and acceleration:

    File Format Description
    build/candidates/*/prompt.json JSON LLM prompt record
    build/candidates/*/response.json.txt Text Raw LLM response
    build/candidates/*/attempt.json JSON Full attempt record
    build/candidates/*/compile.json JSON Compilation result
    build/candidates/*/byte-comparison.json JSON Byte comparison vs original
    build/candidates/*/relocation.json JSON Relocation resolution
    build/candidates/*/report.json JSON Aggregated run report

    Report outputs¶

    Matching reports¶

    File Format Description
    reports/matching/*.json JSON Match verification reports
    reports/matching/*-compile.json JSON Compile gate results
    reports/matching/*-byte-compare.json JSON Byte comparison results
    reports/matching/*-abi.json JSON ABI compatibility results
    reports/matching/*-relink.json JSON Relink gate results

    Functional reports¶

    File Format Description
    reports/functional/*.json JSON Functional validation reports
    reports/functional/*-harness.json JSON Harness execution results
    reports/functional/*-symbolic.json JSON Symbolic execution equality proofs

    Benchmark reports¶

    File Format Description
    reports/benchmarks/*.json JSON Benchmark corpus execution reports

    Convergence reports¶

    File Format Description
    reports/convergence/*.json JSON Project convergence tracking reports

    Reproducibility reports¶

    File Format Description
    reports/reproducibility/*.json JSON Build reproducibility audit reports

    Security reports¶

    File Format Description
    reports/security/*.json JSON Security audit reports

    Reconstruction outputs¶

    File Format Description
    reconstruction/module-assignments.json JSON Module assignment manifest
    reconstruction/ghidra-name-sync.json JSON Ghidra name synchronization record
    reconstruction/runtime-quarantine.json JSON Runtime code quarantine manifest
    reconstruction/build-contract.json JSON Build system descriptor

    Governance outputs¶

    File Format Description
    state/proofs/*.zip ZIP Proof bundles (manifest.json + records.json)
    state/*.changeset.zip ZIP Changeset archives (changeset.json)
    state/workflows/*.json JSON Workflow definition files
    state/counterexamples/* Various Counterexample artifacts with .json metadata

    Test suite outputs¶

    File Format Description
    test-bundle-report.json JSON Test bundle execution summary
    run-report.json JSON Test suite run report

    Capsule outputs¶

    File Format Description
    *.capsule.zip ZIP Function capsules containing capsule-manifest.json and all artifacts

    Diff and integration outputs¶

    File Format Description
    exe-diff-report.json JSON Executable binary diff report
    integration-report.json JSON End-to-end integration test report
    convergence-report.json JSON Project convergence summary

    Note

    All JSON outputs are valid against their corresponding schemas in schemas/. TOML outputs (e.g. target.toml) follow the schema-defined structure for their corresponding JSON representation.
