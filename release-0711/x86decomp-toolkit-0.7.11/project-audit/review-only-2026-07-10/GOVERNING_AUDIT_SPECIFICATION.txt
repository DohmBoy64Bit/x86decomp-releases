Triple-Verification Standard
For every important finding, attempt to verify it through three distinct forms of evidence:

Direct evidence The relevant implementation, configuration, documentation, command definition, test, or exact code path.
Corroborating evidence Call sites, imports, tests, related documentation, schemas, dependency declarations, command registration, examples, or runtime output.
Consistency check Compare the finding against the project’s apparent intended behavior, architecture, naming conventions, public API, CLI help, and related components. If three forms of verification are not available, state exactly what was and was not verified.
