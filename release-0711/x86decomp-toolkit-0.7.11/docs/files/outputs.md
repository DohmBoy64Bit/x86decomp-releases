# Toolkit outputs

This page catalogs the files generated by the x86decomp toolkit during project initialization, analysis, compilation, relinking, evidence management, and reporting.

---

## Project initialization

Files created by `x86decomp project init`:

| File | Format | Description |
|------|--------|-------------|
| `project.json` | JSON | Project configuration (schema v1–3) |
| `analysis/program.json` | JSON | PE program manifest — parsed binary metadata |
| `analysis/host.json` | JSON | Host environment snapshot |
| `analysis/database/analysis.sqlite3` | SQLite | Analysis database |
| `work/tasks.sqlite3` | SQLite | Work queue database |
| `state/project-state.sqlite3` | SQLite | Project state database (v3) |
| `memory/events.jsonl` | JSONL | Memory event ledger (initialized with one event) |
| `target-pack/target.toml` | TOML | Target pack identity (v3) |

---

## Analysis outputs

| File | Format | Description |
|------|--------|-------------|
| `analysis/exports/*.json` | JSON | Exported analysis data |
| `analysis/ghidra/*` | Various | Ghidra project files and data exports |
| `analysis/mcp-transactions/*` | JSON | Recorded MCP requests/responses |
| `analysis/recon/*` | JSON | Reconstruction analysis artifacts |

---

## Compilation outputs

Generated by compiler experiments and matched builds:

| File | Format | Description |
|------|--------|-------------|
| `build/compiler/*.obj` | COFF | Compiled object files |
| `build/compiler/*.asm` | Text | Assembly listings from compilation |
| `build/compiler/*.json` | JSON | Compilation reports (timing, diagnostics) |
| `build/cache/*` | Various | Build cache artifacts |

---

## Relink outputs

Generated by linker reconstruction and relinking:

| File | Format | Description |
|------|--------|-------------|
| `build/relink/manifest.json` | JSON | Relink manifest |
| `build/relink/*.exe` | PE | Relinked executable |
| `build/relink/*.dll` | PE | Relinked library |
| `build/relink/*.json` | JSON | Section layout, COMDAT resolution records |
| `build/relink/*-report.json` | JSON | Relink execution report |
| `build/patches/*` | Binary | Binary patch files |

---

## Function outputs

Each function gets a subdirectory under `functions/` named `pe-rva_XXXXXXXX`:

| File | Format | Description |
|------|--------|-------------|
| `pe-rva_XXXXXXXX/function.json` | JSON | Function packet manifest |
| `pe-rva_XXXXXXXX/*.c` | C | Decompiled C source |
| `pe-rva_XXXXXXXX/*.cpp` | C++ | Decompiled C++ source |
| `pe-rva_XXXXXXXX/*.asm` | Assembly | Byte-form assembly (default) or annotated |
| `pe-rva_XXXXXXXX/references.json` | JSON or JSONL | Cross-reference evidence |
| `pe-rva_XXXXXXXX/compile.json` | JSON | Compilation attempt report |
| `pe-rva_XXXXXXXX/byte-comparison.json` | JSON | Byte comparison results |
| `pe-rva_XXXXXXXX/relocation.json` | JSON | Relocation resolution report |
| `pe-rva_XXXXXXXX/attempt.json` | JSON | Full attempt record including prompt, response, and validation |

## Memory event ledger

| File | Format | Description |
|------|--------|-------------|
| `memory/events.jsonl` | JSONL | Append-only audit log of project operations |

Each event is a JSON object on its own line:

```json
{"actor":"x86decomp","category":"project","summary":"Initialized a native x86 PE decompilation project...","details":{...},"recorded_at":"2026-07-01T12:00:00Z"}
```
